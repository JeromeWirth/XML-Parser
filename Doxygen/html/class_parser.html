void Parser::processEndTag ( )

Es wurde ein schließendes Tag gefunden, und nun wird geprüft, zu welchen öffnenden Tag das Tag gehört.

Da das letzte öffnende Tag als erstes geschlossen werden muss, wird die vector-List, in welchen sich die Elemente befinden, von hinten nach vorne geprüft. Wenn man dann auf ein Element gestoßen ist, welches noch nicht Valid ist, also noch kein schliessendes Tag hat, und der Name der beiden Tags gleich ist, wird das schliessenden Tag, diesem Tag zugewiesen.

void Parser::processInput ( )

Die Funktion liest einen String solange ein, bis er die größe 0 hat. Anhand des ersten Zeichen des Strings wird entschieden, wie der String weiter verarbeitet werden soll.

Sollte das erste Zeichen ein Leerzeichen oder Tab sein, so wird dieser String erstmal normalisiert und dann weiter verarbeitet.

void Parser::processStartTag ( string  start)

Die Funktion analysiert den Starttag, wenn processInput() eine öffnende eckige Klammer gefunden hat, teilt die funktion den String auf und übermittelt den String in dem sich nur der Starttag (also alles zwischen '<' und '>') befindet, an die Funktion processStartTag. Wenn der String fertig verarbeitet wurde, wird er von dem ursprünglichen String subtrahiert.

Parameters
startDer String der den gesamten Starttag beinhaltet
void Parser::processSystemTag ( )

Die funktion analysiert den Tag auf Zeichen, welche auf Kommentare und Systemanweisungen hindeuten, dann wird der String ignoriert, oder falls erkannt wurde das es ein Syntaxfehler in dem Tag gibt, eine Fehlermeldung ausgegeben.

void Parser::processTagValue ( )

Wenn ein Zeichen gefunden wurde, welches keine öffnende oder schliessende Klammer ist, wird davon ausgegangen, dass es sich um den Elementwert handel.

Dazu wird geprüft, ob sich in dem String noch eine klammer befindet oder ob nur noch Text in dem string steht, nach der verarbeitung wird dieser String wieder vom ursprünglichen String abgezogen.
